# Internal documentation

This folder contains internal documentation so that you can use LabConstrictor functionalaties smoothly. If you want to have your own documentation for your project we recommend you to create and upload it to a `docs` folder in your repository root.

On this internal documentation you will find guidelines on how to upload and update notebooks to your repository, how to create a executable files or how can you upload custom code to be used on your notebooks.

- [How to upload and update notebooks?](notebook_upload.md)
- [How to create executable files?](executable_creation.md)
- [How to upload custom code?](custom_code_upload.md)
- [How to check the automatic workflow status?](workflow_status.md)

You will also find the original README.md file before initializing your notebook [here](labconstrictor.md).
